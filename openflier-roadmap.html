OpenFlier Roadmap - 2023

In this article, we will give a brief introduction to OpenFlier, and what we would like to achive before Sep. 2023.

What is OpenFlier

OpenFlier is a successor to Zyflier. It is also the open source version of Zyflier.

Why OpenFlier

OpenFlier is designed to be open source, extensible and easy to maintain. By exposing more underlying interfaces and options to the user, it can be quickly modified to be compatible with the lastest version of Cloudsnote. Also, being open source means that we can get support from more people.

Roadmap

Stage 1: Implement basic features

- UDP server, FTP server, MQTT server
- Connection & Validation

Stage 2: Implement screenshot features

This stage will be the most important stage. We will rewrite most of the code to ensure stability.
- Basic screenshots
- SpecialChannels

Stage 3: Implement Configurator

We will combine Zyflier and EzyConfigurator together in OpenFlier to support configuration hot reload.
Also, we will be storing configuration in Json format.

Stage 4: Implement extension features

By using an extensible architecture, OpenFlier will enable more developers to join in the development.
